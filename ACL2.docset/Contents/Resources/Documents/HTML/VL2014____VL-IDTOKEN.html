Raw constructor for vl-idtoken-p structures.

Syntax:

(vl-idtoken etext name)

This is the lowest-level constructor for vl-idtoken-p structures. It simply conses together a structure with the specified fields.

Note: It's generally better to use macros like make-vl-idtoken or change-vl-idtoken instead. These macros lead to more readable and robust code, because you don't have to remember the order of the fields.

The vl-idtoken-p structures we create here are just constructed with ordinary cons. If you want to create honsed structures, see honsed-vl-idtoken instead.

Definition

This is an ordinary constructor function introduced by defaggregate.

Function: vl-idtoken

(defun vl-idtoken (etext name)
       (declare (xargs :guard (and (and (vl-echarlist-p etext)
                                        (consp etext)
                                        (true-listp etext))
                                   (stringp name))))
       (cons :vl-idtoken (cons etext name)))
