Load the state of the wallet from a file.

Signature
(load-stat state) → (mv error? stat? state)
Returns
error? — Type (maybe-command-error-p error?).
stat? — Type (maybe-statp stat?).

The name suffix -stat of this function is consistent with the type name stat. It emphasizes the wallet state, as opposed to the ACL2 state.

serialize-read throws a hard error upon failure. This may acceptable if the shell script that calls the wallet can catch that and turn into a more user-oriented error message.

Definitions and Theorems

Function: load-stat

(defun load-stat (state)
       (declare (xargs :stobjs (state)))
       (declare (xargs :guard t))
       (b* (((mv stat? state)
             (serialize-read *stat-filepath*))
            ((unless (and (statp stat?) (stat-wfp stat?)))
             (mv (command-error-state-file-malformed)
                 nil state)))
           (mv nil stat? state)))
