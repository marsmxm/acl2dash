Modifying constructor for literal-result-err structures.

Syntax
(change-literal-result-err x 
                           [:get <get>])

This is an often useful alternative to make-literal-result-err.

We construct a new literal-result-err structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-literal-result-err

(defmacro
     change-literal-result-err (x &rest args)
     (std::change-aggregate 'literal-result-err
                            x
                            args '((:get . literal-result-err->get))
                            'change-literal-result-err
                            'nil))
