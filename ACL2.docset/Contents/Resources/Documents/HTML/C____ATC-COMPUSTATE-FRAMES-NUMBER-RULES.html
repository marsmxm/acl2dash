Rules about compustate-frames-number.

The first four theorems about compustate-frames-number serve to discharge the hypotheses about it being not 0 in some of the other theorems. These are all immediately discharged, because the functions used to represent the symbolic execution states satisfy that condition (by design, for add-var and update-var).

The last theorem serves to skip over update-array calls.

Definitions and Theorems

Theorem: compustate-frames-number-of-add-frame-not-zero

(defthm
     compustate-frames-number-of-add-frame-not-zero
     (not (equal (compustate-frames-number (add-frame fun compst))
                 0)))

Theorem: compustate-frames-number-of-enter-scope-not-zero

(defthm compustate-frames-number-of-enter-scope-not-zero
        (not (equal (compustate-frames-number (enter-scope compst))
                    0)))

Theorem: compustate-frames-number-of-add-var-not-zero

(defthm
     compustate-frames-number-of-add-var-not-zero
     (not (equal (compustate-frames-number (add-var var val compst))
                 0)))

Theorem: compustate-frames-number-of-update-var-not-zero

(defthm
  compustate-frames-number-of-update-var-not-zero
  (not (equal (compustate-frames-number (update-var var val compst))
              0)))

Theorem: compustate-frames-number-of-update-array

(defthm
   compustate-frames-number-of-update-array
   (equal (compustate-frames-number (update-array ptr array compst))
          (compustate-frames-number compst)))
