Returns an svex representing the sign- or zero-extension of x at the given width.

Signature
(svex-extend type width x) → sv
Arguments
type — Guard (vl-exprsign-p type).
width — Guard (natp width).
x — Guard (sv::svex-p x).
Returns
sv — Type (sv::svex-p sv).

We don't have to extend/truncate operands when translating VL expressions like a & b, but we do need to do it to the inputs of expressions like a < b, to chop off any garbage in the upper bits.

Definitions and Theorems

Function: svex-extend

(defun svex-extend (type width x)
       (declare (xargs :guard (and (vl-exprsign-p type)
                                   (natp width)
                                   (sv::svex-p x))))
       (let ((__function__ 'svex-extend))
            (declare (ignorable __function__))
            (if (vl-exprsign-equiv type :vl-signed)
                (sv::svex-signx width x)
                (sv::svex-zerox width x))))
