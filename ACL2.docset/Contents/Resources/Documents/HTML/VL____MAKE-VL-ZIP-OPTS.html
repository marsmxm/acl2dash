Constructor macro for vl-zip-opts-p structures.

Syntax:

(make-vl-zip-opts [:help <help>] 
                  [:readme <readme>] 
                  [:name <name>] 
                  [:output <output>] 
                  [:start-files <start-files>] 
                  [:plusargs <plusargs>] 
                  [:search-path <search-path>] 
                  [:include-dirs <include-dirs>] 
                  [:search-exts <search-exts>] 
                  [:defines <defines>] 
                  [:edition <edition>] 
                  [:strict <strict>] 
                  [:mem <mem>])

This is our preferred way to construct vl-zip-opts-p structures. It simply conses together a structure with the specified fields.

This macro generates a new vl-zip-opts-p structure from scratch. See also change-vl-zip-opts, which can "change" an existing structure, instead.

The vl-zip-opts-p structures we create here are just constructed with ordinary cons. If you want to create honsed structures, see make-honsed-vl-zip-opts instead.

Definition

This is an ordinary make- macro introduced by defaggregate.

Macro: make-vl-zip-opts

(defmacro make-vl-zip-opts (&rest args)
          (std::make-aggregate 'vl-zip-opts
                               args
                               '((:help)
                                 (:readme)
                                 (:name . "design")
                                 (:output . "design.vlzip")
                                 (:start-files)
                                 (:plusargs)
                                 (:search-path)
                                 (:include-dirs quote ("."))
                                 (:search-exts quote ("sv" "v"))
                                 (:defines)
                                 (:edition . :system-verilog-2012)
                                 (:strict)
                                 (:mem . 4))
                               'make-vl-zip-opts
                               nil))
