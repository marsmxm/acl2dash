All the JSON grammar rules, including the referenced ABNF core rules.

These are obtained by plugging the core rules into the IMAP rules.

These rules are well-formed and closed.

These rules generate terminal strings consisting only of Unicode codes, i.e. natural numbers between 0 and x10FFFF. Proving this by execution would take a long time due to the specification-appropriate but execution-inefficient definition of rulelist-in-termset-p and to the relatively large size of the range of natural numbers. Thus, we prove the theorems (quickly) by disabling the executable counterpart of integers-from-to and by using library theorems that relate integers-from-to to in and list-in.

We use add-const-to-untranslate-preprocess to keep this constant unexpanded in output.

Definition: *all-json-grammar-rules*

(defconst *all-json-grammar-rules*
          (plug-rules *json-grammar-rules* *core-rules*))
