Case macro for the different kinds of proof-list-outcome structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a proof-list-outcome structure, or to split into cases based on its type.

Short Form

In its short form, proof-list-outcome-case allows you to safely check the type of a proof-list-outcome structure. For example:

(proof-list-outcome-case x :assertions)

is essentially just a safer alternative to writing:

(equal (proof-list-outcome-kind x) :assertions)

Why is using proof-list-outcome-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :assertions is a valid kind of proof-list-outcome structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :assertions to something else. It also means you get no help if you just make a typo when writing the :assertions symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, proof-list-outcome-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(proof-list-outcome-case x
  :assertions ...
  :fail ...
  :error ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :assertions case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a assertions b* binder.
