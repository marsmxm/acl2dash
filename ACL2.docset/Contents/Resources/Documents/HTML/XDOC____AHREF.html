Construct an XDOC tree for a string of the form <a href="...">...</a>.

This is a fairly common pattern. This XDOC constructor obviates the need to type :href, compared to the primitive constructor a: the first argument is always the URL, and the remaining arguments form the text.

Macro: ahref

(defmacro ahref (href &rest text)
          (cons 'a
                (cons ':href (cons href text))))
