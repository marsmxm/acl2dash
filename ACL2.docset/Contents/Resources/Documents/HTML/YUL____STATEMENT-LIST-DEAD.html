Eliminate dead code in lists of statements.

Signature
(statement-list-dead stmts afterp) → new-stmt
Arguments
stmts — Guard (statement-listp stmts).
afterp — Guard (booleanp afterp).
Returns
new-stmt — Type (statement-listp new-stmt).

This function takes a flag saying whether the statements being examined come after (hence the name) a leave, break, or continue statement. The flag is initially false (set by block-dead), and becomes true when we encounter one of those three statements. So long as the flag is false, we keep the statements, recursively transforming them. When the flag is true, we only keep function definitions, recursively transforming them.

Note that we cannot simply stop when we encounter a leave, break, or continue and discard everything after it, because we still need to keep the function definitions.
