Case macro for the different kinds of vl-paramargs structures.

This is an fty sum-type case macro, typically introduced by defflexsum or deftagsum. It allows you to safely check the type of a vl-paramargs structure, or to split into cases based on its type.

Short Form

In its short form, vl-paramargs-case allows you to safely check the type of a vl-paramargs structure. For example:

(vl-paramargs-case x :vl-paramargs-named)

is essentially just a safer alternative to writing:

(equal (vl-paramargs-kind x) :vl-paramargs-named)

Why is using vl-paramargs-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :vl-paramargs-named is a valid kind of vl-paramargs structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :vl-paramargs-named to something else. It also means you get no help if you just make a typo when writing the :vl-paramargs-named symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, vl-paramargs-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(vl-paramargs-case x
  :vl-paramargs-named ...
  :vl-paramargs-plain ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :vl-paramargs-named case, you can use defprod-style foo.bar style accessors for x without having to explicitly add a vl-paramargs-named b* binder.
