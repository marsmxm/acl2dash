Generate a C expression from an ACL2 term that must be an expression term returning a boolean.

Signature
(atc-gen-expr-bool term inscope fn ctx state) 
   
(mv erp expr state)
Arguments
term Guard (pseudo-termp term).
inscope Guard (atc-symbol-type-alist-listp inscope).
fn Guard (symbolp fn).
ctx Guard (ctxp ctx).
Returns
expr Type (exprp expr).

At the same time, we check that the term is an expression term returning a boolean, as described in the user documentation.

If the term is a call of not, and, or or, we recursively translate the arguments, which must be an expression term returning a boolean, and we construct a logical expression with the corresponding C operators.

If the term is a call of boolean-from-<type>, we call the mutually recursive function that translates the argument, which must be an expression term returning a C value, to an expression, which we return.

In all other cases, we fail with an error. The term is not an expression term returning a C value. We could extend this code to provide more information to the user at some point.

As in atc-gen-expr-cval-pure, we perform C type checks on the ACL2 terms. See atc-gen-expr-cval-pure for an explanation.
