Representation of wire (net) types, i.e., resolution function.

Note that Verilog/SystemVerilog net types like wire and wor essentially govern how multiple drivers get resolved, and are entirely distinct from the normal idea of a data type. See vl-vardecl for additional discussion.

This is an ordinary defenum.

Function: vl-nettypename-p

(defun vl-nettypename-p (x)
       (declare (xargs :guard t))
       (or (eq x ':vl-wire)
           (eq x ':vl-tri)
           (eq x ':vl-supply0)
           (eq x ':vl-supply1)
           (eq x ':vl-triand)
           (eq x ':vl-trior)
           (eq x ':vl-tri0)
           (eq x ':vl-tri1)
           (eq x ':vl-trireg)
           (eq x ':vl-uwire)
           (eq x ':vl-wand)
           (eq x ':vl-wor)))

(defthm type-when-vl-nettypename-p
