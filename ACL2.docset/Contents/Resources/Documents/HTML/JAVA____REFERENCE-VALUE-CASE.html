Case macro for the different kinds of reference-value structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a reference-value structure, or to split into cases based on its type.

Short Form

In its short form, reference-value-case allows you to safely check the type of a reference-value structure. For example:

(reference-value-case x :pointer)

is essentially just a safer alternative to writing:

(equal (reference-value-kind x) :pointer)

Why is using reference-value-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :pointer is a valid kind of reference-value structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :pointer to something else. It also means you get no help if you just make a typo when writing the :pointer symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, reference-value-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(reference-value-case x
  :pointer ...
  :null ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :pointer case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a pointer b* binder.
