Representation for a stack of scopes, providing read-only access to elaboration info.

Elabscopes are useful for providing "read-only" (applicative) access to elaboration information, e.g. once elaboration is over or once all elaboration info necessary for some purpose has been computed. For a structure that supports adding in new elaboration info, use an elabindex stobj.

Interface

  • vl-elabscopes-init creates a new elabscopes at the design level.
  • vl-elabscopes-push-scope enters a scope inside the current one.
  • vl-elabscopes-pop pops out n scope levels.
  • vl-elabscopes-root pops out to the root (global) scope.
  • vl-elabscopes-traverse enters the scope specified by a traversal (returned by certain utilies such as vl-follow-scopeexpr).
  • vl-elabscopes-item-info gets the elaboration info for a local item, if it exists.

Except for vl-elabscopes-init, these each take a keyword argument :allow-empty which determines whether a hard error occurs if the scopes are empty.
