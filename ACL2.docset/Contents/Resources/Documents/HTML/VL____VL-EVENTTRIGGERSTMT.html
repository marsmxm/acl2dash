Representation of an event trigger.

This is a product type, introduced by deftagsum in support of vl-stmt.

Fields
id — vl-expr
Typically a name like foo and bar, but may instead be a hierarchical identifier.
atts — vl-atts
Any (* foo, bar = 1*) style attributes associated with this statement.

Event trigger statements are used to explicitly trigger named events. They are discussed in Section 9.7.3 and looks like this:

-> foo;
-> bar[1][2][3];  // Weirdly permitted in Verilog-2005 but
                  // not in SystemVerilog-2012.

We put any indexing operations into the id expression.
