Check a variable against a symbol table.

Signature
(atc-check-var var inscope) → (mv type? innermostp errorp)
Arguments
var — Guard (symbolp var).
inscope — Guard (atc-symbol-type-alist-listp inscope).
Returns
type? — Type (type-optionp type?).
innermostp — Type (booleanp innermostp).
errorp — Type (booleanp errorp).

If the variable is in the symbol table, we return its type, along with a flag indicating whether the variable is in the innermost scope. If the symbol table contains a different variable with the same symbol name, we return an indication of error; this is because ACL2 variables represent C variables whose names are just the symbol names of the ACL2 variables, which therefore must be distinct for different ACL2 variables.

It is an invariant that all the variables in the symbol table have distinct symbol names.

Function: atc-check-var-aux

(defun
 atc-check-var-aux
 (var inscope innermostp)
 (declare (xargs :guard (and (symbolp var)
                             (atc-symbol-type-alist-listp inscope)
                             (booleanp innermostp))))
 (let
  ((__function__ 'atc-check-var-aux))
  (declare (ignorable __function__))
  (b*
     (((when (endp inscope)) (mv nil nil nil))
      (scope (car inscope))
      (type? (cdr (assoc-eq var (atc-symbol-type-alist-fix scope))))
      ((when type?) (mv type? innermostp nil))
      ((when (member-equal (symbol-name var)
                           (symbol-name-lst (strip-cars scope))))
       (mv nil nil t)))
     (atc-check-var-aux var (cdr inscope)
                        nil))))
