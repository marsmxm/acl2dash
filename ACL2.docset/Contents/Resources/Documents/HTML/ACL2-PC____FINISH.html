(macro) require completion of instructions; save error if inside :hints

Example:
(finish induct prove bash)

General Form:
(finish &rest instructions)

Run the indicated instructions, stopping at the first failure. If there is any failure, or if any new goals are created and remain at the end of the indicated instructions, then consider the call of finish to be a failure. See proof-builder-commands and see ACL2-pc::sequence for a discussion of the notion of ``failure'' for interactive proof-builder commands.
