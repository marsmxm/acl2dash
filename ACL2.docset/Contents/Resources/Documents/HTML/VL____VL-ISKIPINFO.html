Information about the multiple-include status of an individual file.

This is a product type introduced by defprod.

Fields
controller — maybe-string
NIL if this file does not have a proper include guard, or the name of the controlling define otherwise.
misses — vl-locationlist
For evaluating multiple-include optimization effectiveness. This is a list of places where this file is `included and is actually being loaded from disk. For files that follow proper include-guard discipline, this should end up being a singleton list with only the initial `include of the file. For files that don't follow the include-guard discipline but are included multiple times, it may be a longer list.
hits — vl-locationlist
For evaluating multiple-include optimization effectiveness. This is a list of places where this file is being `included from but where we don't bother reloading it thanks to MI optimization.
len — natp
Length of this file in characters. Possibly of interest for evaluating multiple-include effectiveness.
