The theorem's formula is

(implies (domain a)
         (equal (new x1 ... xn a)
                combine<a,(old x1 ... xn)>))

if the variant is :monoid, :monoid-alt, or :assoc, while it is

(equal (new x1 ... xn a)
       combine<a,(old x1 ... xn)>)

if the variant is :assoc-alt.

The hints follow the proofs in the design notes. Note that combine-right-identity-thm? is nil iff the :combine-right-identity applicability condition is absent, which happens exactly when the :variant input to the transformation is :assoc or :assoc-alt. In this case, no instance of that applicability condition is used in the proof.

                                      'nil))))))

          (cons
           ':instance

            combine-associativity-thm
            (cons
             ':extra-bindings-ok

                        'nil)))))))

           combine-right-identity-thm?

            ':instance

                                    (cons a 'nil))
                              'nil))))))

          (cons
           ':instance
           (cons
              domain-of-old-name
              (cons ':extra-bindings-ok

         (cons
          '"Goal"
          (cons
           ':in-theory
           (cons
            (cons
             'quote

              (cons
                old-unnorm-name
                (cons new-unnorm-name
                      (cons (cons ':induction (cons new-name$ 'nil))
                            'nil)))
              'nil))
            (cons ':induct
                  (cons (cons new-name$ new-formals)
                        'nil)))))
         (cons
          (cons
           'quote
           (cons

             ':use

                domain-of-nonrec-thm
