Rules related to not.

ATC currently allows the logical negation operator ! to be represented via not only the lognot-<type> functions, but also not, with appropriate conversions with booleans as needed. That is, an ACL2 target function, from which C code is generate, may represent logical negation in two different ways. The dynamic semantics of C, during symbolic execution, turns expressions with the logical negation operator into one form. Thus, we need rules to normalize one of the two forms into the other.

We choose to normalize things to use not. Thus, we add rules saying that (boolean-from-<type> (lognot-<type> x)) becomes (not (boolean-from-<type> x)).

Because these rules may be applied to terms that result from execution and that must be checked not to be errors, we also add a rule saying that a call of not is not an error. Otherwise, terms of the form (errorp (not ...)) arise during symbolic execution and cause proofs to fail.

It may be better to change ATC to require a unique representation of the logical negation operator, and avoid these rules altogether. This approach will be considered in the future.

Theorem: boolean-from-uchar-of-lognot-uchar

(defthm boolean-from-uchar-of-lognot-uchar
        (equal (boolean-from-uchar (lognot-uchar x))
               (not (boolean-from-uchar x))))
