Case macro for the different kinds of obligation-hyp structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a obligation-hyp structure, or to split into cases based on its type.

Short Form

In its short form, obligation-hyp-case allows you to safely check the type of a obligation-hyp structure. For example:

(obligation-hyp-case x :condition)

is essentially just a safer alternative to writing:

(equal (obligation-hyp-kind x) :condition)

Why is using obligation-hyp-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :condition is a valid kind of obligation-hyp structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :condition to something else. It also means you get no help if you just make a typo when writing the :condition symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, obligation-hyp-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(obligation-hyp-case x
  :condition ...
  :binding ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :condition case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a condition b* binder.
