Partition a list of vl-portdecls by name.

Signature
(vl-filter-portdecls names x) → (mv named unnamed)
Arguments
names — Names to filter with.
    Guard (string-listp names).
x — List to filter.
    Guard (vl-portdecllist-p x).

The only reason to use this function is efficiency. Logically, named is equal to vl-keep-portdecls and unnamed is equal to vl-delete-portdecls. We leave this function enabled and would think it odd to ever prove a theorem about it.

Definitions and Theorems

Function: vl-filter-portdecls

(defun
 vl-filter-portdecls (names x)
 (declare (xargs :guard (and (string-listp names)
                             (vl-portdecllist-p x))))
 (let
  ((__function__ 'vl-filter-portdecls))
  (declare (ignorable __function__))
  (mbe
      :logic (mv (vl-keep-portdecls names x)
                 (vl-delete-portdecls names x))
      :exec (b* (((when (atom names))
                  (mv nil (list-fix x)))
                 ((when (atom x)) (mv nil nil))
                 (fal (make-lookup-alist names))
                 ((local-stobjs nrev nrev2)
                  (mv yes no nrev nrev2))
                 ((mv nrev nrev2)
                  (vl-fast-filter-portdecls names fal x nrev nrev2))
                 (- (fast-alist-free fal))
                 ((mv yes nrev) (nrev-finish nrev))
                 ((mv no nrev2) (nrev-finish nrev2)))
                (mv yes no nrev nrev2)))))

(defthm
   vl-fast-filter-portdecls-removal-0
   (equal (mv-nth 0
                  (vl-fast-filter-portdecls names fal x nrev nrev2))
