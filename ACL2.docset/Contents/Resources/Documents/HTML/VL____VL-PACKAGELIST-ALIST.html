Extend an alist by binding the names of vl-packages to their definitions.

Signature
(vl-packagelist-alist x acc) → alist
Arguments
x — Guard (vl-packagelist-p x).
Returns
alist — Type (equal (vl-package-alist-p alist) (vl-package-alist-p acc)).

This can be used as an alternative to vl-find-package when you need to perform a lot of lookups.

Definitions and Theorems

Function: vl-packagelist-alist

(defun vl-packagelist-alist (x acc)
       (declare (xargs :guard (vl-packagelist-p x)))
       (let ((__function__ 'vl-packagelist-alist))
            (declare (ignorable __function__))
            (if (atom x)
                acc
                (cons (cons (vl-package->name (car x))
                            (vl-package-fix (car x)))
                      (vl-packagelist-alist (cdr x) acc)))))

     (syntaxp (not (equal acc ''nil)))

Theorem: lookup-in-vl-packagelist-alist-fast

(defthm
    lookup-in-vl-packagelist-alist-fast
    (implies
         (stringp name)
         (equal (hons-assoc-equal name (vl-packagelist-alist x nil))
                (let ((val (vl-find-package name x)))
                     (and val (cons name val))))))
