Modifying constructor for expression-cond structures.

Syntax
(change-expression-cond x 
                        [:branches <branches>])

This is an often useful alternative to make-expression-cond.

We construct a new expression-cond structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-expression-cond

(defmacro
   change-expression-cond (x &rest args)
   (std::change-aggregate 'expression-cond
                          x args
                          '((:branches . expression-cond->branches))
                          'change-expression-cond
                          'nil))
