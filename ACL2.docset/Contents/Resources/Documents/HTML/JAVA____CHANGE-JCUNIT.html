Modifying constructor for jcunit structures.

Syntax
(change-jcunit x 
               [:package? <package?>] 
               [:imports <imports>] 
               [:types <types>])

This is an often useful alternative to make-jcunit.

We construct a new jcunit structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-jcunit

(defmacro change-jcunit (x &rest args)
          (std::change-aggregate 'jcunit
                                 x args
                                 '((:package? . jcunit->package?)
                                   (:imports . jcunit->imports)
                                   (:types . jcunit->types))
                                 'change-jcunit
                                 'nil))
