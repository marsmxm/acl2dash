Case macro for the different kinds of member-type-list-option structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a member-type-list-option structure, or to split into cases based on its type.

Short Form

In its short form, member-type-list-option-case allows you to safely check the type of a member-type-list-option structure. For example:

(member-type-list-option-case x :some)

is essentially just a safer alternative to writing:

(equal (member-type-list-option-kind x) :some)

Why is using member-type-list-option-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :some is a valid kind of member-type-list-option structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :some to something else. It also means you get no help if you just make a typo when writing the :some symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, member-type-list-option-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(member-type-list-option-case x
  :some ...
  :none ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :some case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a some b* binder.
