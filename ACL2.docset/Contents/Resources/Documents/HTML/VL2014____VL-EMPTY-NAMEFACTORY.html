(vl-empty-namefactory) creates an empty name factory without a module.

Signature
(vl-empty-namefactory)  nf
Returns
nf Type (vl-namefactory-p nf).

Usually you should use vl-starting-namefactory instead; vl-starting-namefactory automatically regards all of the names in the module as used, whereas vl-empty-namefactory regards no names as used.

On the other hand, vl-empty-namefactory may be useful when you are generating modules from scratch and, hence, don't have a module to give to vl-starting-namefactory yet.
