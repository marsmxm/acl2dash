Fixtype of statement types.

This is a product type introduced by fty::defprod.

Fields
return-types — type-set
variables — var-table
Additional Requirements

The following invariant is enforced on the fields:

(not (empty return-types))

Here we use the word ``type'' in a broad sense, namely to describe the information inferred by the static semantics about a statement (or block item or block). The information consists of:

  • A non-empty set of types that describes the possible values returned by the statement. These are determined by the return statements; in the presence of conditionals, the possible types in the two branches are merged (i.e. unioned). The type void is used to describe statements that do not return a value, but instead transfer control to the next statement (if any).
  • A possibly updated variable table. This is updated by block items that are declarations. We actually only need to return possibly updated variable tables from the ACL2 function check-block-item; the ACL2 functions check-stmt and check-block-item-list could just return a set of optional types (see above). However, for uniformity we have all three functions return also a possibly updated variable table.
