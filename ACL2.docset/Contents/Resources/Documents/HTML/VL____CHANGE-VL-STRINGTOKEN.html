A copying macro that lets you create new vl-stringtoken-p structures, based on existing structures.

Syntax:

(change-vl-stringtoken x 
                       [:etext <etext>] 
                       [:expansion <expansion>] 
                       [:breakp <breakp>])

This is a sometimes useful alternative to make-vl-stringtoken. It constructs a new vl-stringtoken-p structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by defaggregate.

Macro: change-vl-stringtoken

(defmacro
   change-vl-stringtoken (x &rest args)
   (std::change-aggregate 'vl-stringtoken
                          x args
                          '((:etext . vl-stringtoken->etext)
                            (:expansion . vl-stringtoken->expansion)
                            (:breakp . vl-stringtoken->breakp))
                          'change-vl-stringtoken
                          'remake-vl-stringtoken))
