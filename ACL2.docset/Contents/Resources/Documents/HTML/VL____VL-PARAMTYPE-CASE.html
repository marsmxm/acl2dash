Case macro for the different kinds of vl-paramtype structures.

This is an fty sum-type case macro, typically introduced by defflexsum or deftagsum. It allows you to safely check the type of a vl-paramtype structure, or to split into cases based on its type.

Short Form

In its short form, vl-paramtype-case allows you to safely check the type of a vl-paramtype structure. For example:

(vl-paramtype-case x :vl-implicitvalueparam)

is essentially just a safer alternative to writing:

(equal (vl-paramtype-kind x) :vl-implicitvalueparam)

Why is using vl-paramtype-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :vl-implicitvalueparam is a valid kind of vl-paramtype structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :vl-implicitvalueparam to something else. It also means you get no help if you just make a typo when writing the :vl-implicitvalueparam symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, vl-paramtype-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(vl-paramtype-case x
  :vl-implicitvalueparam ...
  :vl-explicitvalueparam ...
  :vl-typeparam ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :vl-implicitvalueparam case, you can use defprod-style foo.bar style accessors for x without having to explicitly add a vl-implicitvalueparam b* binder.
