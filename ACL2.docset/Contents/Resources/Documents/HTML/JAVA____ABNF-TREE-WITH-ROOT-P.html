Recognize terminated ABNF trees whose root is the given rule name, for the Java grammar.

Signature
(abnf-tree-with-root-p tree rulename) → yes/no
Arguments
rulename — Guard (stringp rulename).
Returns
yes/no — Type (booleanp yes/no).

This is a useful abbreviation for a more verbose conjunction of ABNF predicates with more verbose arguments.

The tree is terminated, i.e. it has natural numbers at its leaves, not rule names. These natural numbers are Java Unicode characters, because the grammar satisfies abnf::rulelist-in-termset-p for the set (integers-from-to 0 #xffff) as proved here. To prove that the string at the leaves satisfies unicode-listp, we proceed as follows:

  1. We disable integers-from-to and its executable counterpart, so that (integers-from-to 0 #xffff) does not get expanded.
  2. We prove that membership in (integers-from-to 0 #xffff) is equivalent to unicodep (via a library theorem that rewrites the membership assertion into integer inequalities), and then lift that to list-in and unicode-listp.
  3. We prove that abnf-tree-with-root-p implies abnf::string-parsablep, and then abnf::terminal-string-for-rules-p.
  4. We combine the facts from the previous two steps with the abnf::rulelist-in-termset-p theorem about the grammar and with the ABNF library theorem abnf::terminal-strings-in-termset-when-rules-in-termset, which relates the meta-syntactic property abnf::rulelist-in-termset-p with membership in the set of terminals, to obtain the final theorem.

Function: abnf-tree-with-root-p

(defun abnf-tree-with-root-p (tree rulename)
       (declare (xargs :guard (stringp rulename)))
       (and (abnf::treep tree)
            (abnf::tree-terminatedp tree)
            (abnf::tree-match-element-p
                 tree
                 (abnf::element-rulename (abnf::rulename rulename))
                 *grammar*)))
