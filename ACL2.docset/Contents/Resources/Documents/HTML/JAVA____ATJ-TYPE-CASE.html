Case macro for the different kinds of atj-type structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a atj-type structure, or to split into cases based on its type.

Short Form

In its short form, atj-type-case allows you to safely check the type of a atj-type structure. For example:

(atj-type-case x :acl2)

is essentially just a safer alternative to writing:

(equal (atj-type-kind x) :acl2)

Why is using atj-type-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :acl2 is a valid kind of atj-type structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :acl2 to something else. It also means you get no help if you just make a typo when writing the :acl2 symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, atj-type-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(atj-type-case x
  :acl2 ...
  :jprim ...
  :jprimarr ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :acl2 case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a acl2 b* binder.
