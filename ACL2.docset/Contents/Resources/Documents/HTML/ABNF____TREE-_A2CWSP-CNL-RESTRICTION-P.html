Restrictions on the (non-starting) (*c-wsp c-nl) trees generated by the parser.

Signature
(tree-*cwsp-cnl-restriction-p tree) → yes/no
Arguments
tree — Guard (treep tree).
Returns
yes/no — Type (booleanp yes/no).

In order for a (*c-wsp c-nl) tree not to start with WSP, either its *c-wsp repetition must be empty or its first c-wsp subtree must satisfy the restriction captured by tree-cwsp-restriction-p.
