Just a different way to define <= for sexpr alists.

The definition of 4v-sexpr-alist-<= compares the actual sexprs bound in the alists. Here, instead, we first evaluate all the sexprs in the alist, then compare their results using our simple 4v-alist-<= comparison for four-valued alists.

Either definition is equivalent, as established by this theorem, which is ordinarily disabled.

Theorem: 4v-sexpr-alist-<=-is-alt

(defthm 4v-sexpr-alist-<=-is-alt
        (iff (4v-sexpr-alist-<= a b)
             (4v-sexpr-alist-<=-alt a b)))
