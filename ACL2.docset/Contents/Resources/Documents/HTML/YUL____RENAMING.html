Additional Requirements

The following invariant is enforced on the fields:

(and (no-duplicatesp-equal (strip-cars list)) 
     (no-duplicatesp-equal (strip-cdrs list)))

These are alists from identifiers to identifiers that have unique keys and unique values, which we treat as lists of pairs rather than as alist as such (see discussion in renaming-variables). We wrap the alist into a one-component product type and we add constraints for key and value uniqueness.

We use this to capture variable renamings, but it can be also used for function renamings. We will put this into a more general place and also use this for function renamings.
