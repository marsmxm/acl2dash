Push the current function onto the stack of an error result.

This is useful when receiving an error result from a called function, to add the caller to the stack before propagating the error. This addition is handled automatically when using patbind-ok, but when that binder cannot be used for some reason, then this err-push macro is handy.

This assumes that __function__ is bound to the current function name, which is automatically the case when using define.

Macro: err-push

(defmacro
 err-push (error)
 (cons
  'b*
  (cons
   (cons
    (cons 'error (cons error 'nil))
    (cons
     (cons 'stack
           (cons (cons 'resulterr->stack
                       (cons 'error 'nil))
                 'nil))
     (cons
          (cons 'new-stack
                (cons (cons 'cons
                            (cons '__function__ (cons 'stack 'nil)))
                      'nil))
          'nil)))
   (cons (cons 'change-resulterr
               (cons 'error
                     (cons ':stack (cons 'new-stack 'nil))))
         'nil))))
