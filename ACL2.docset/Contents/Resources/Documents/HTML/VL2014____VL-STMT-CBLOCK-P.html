Recognizer for very restricted statements that we (potentially) support in combinational always blocks; i.e., the valid guts for always @@(a or b or ...) guts.

Signature
(vl-stmt-cblock-p x)  *
Arguments
x Guard (vl-stmt-p x).

We expect to see accept:

  • Null statements
  • Blocking assignments with resolved widths
  • If/else statements
  • Simple begin/end blocks (with, e.g., no names or declarations)

We don't try to support, e.g., case statements, since they can be dealt with separately; see caseelim. We don't require that the if have been flattened or anything like that.
