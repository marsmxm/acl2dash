Case macro for the different kinds of neteval-sigordering structures.

This is an fty sum-type case macro, typically introduced by defflexsum or deftagsum. It allows you to safely check the type of a neteval-sigordering structure, or to split into cases based on its type.

Short Form

In its short form, neteval-sigordering-case allows you to safely check the type of a neteval-sigordering structure. For example:

(neteval-sigordering-case x :segment)

is essentially just a safer alternative to writing:

(equal (neteval-sigordering-kind x) :segment)

Why is using neteval-sigordering-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :segment is a valid kind of neteval-sigordering structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :segment to something else. It also means you get no help if you just make a typo when writing the :segment symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, neteval-sigordering-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(neteval-sigordering-case x
  :segment ...
  :remainder ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :segment case, you can use defprod-style foo.bar style accessors for x without having to explicitly add a segment b* binder.
