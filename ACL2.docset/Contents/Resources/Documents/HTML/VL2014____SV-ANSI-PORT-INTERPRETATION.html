SystemVerilog-2012 rules for determining port kind/type/direction for ANSI ports (Section 23.2.2.3).

SystemVerilog has some tricky rules for how ANSI port lists are interpreted. For instance, in a module like:

module foo (output o,
            input logic [3:0] a, b, c) ;
  ...
endmodule

The input logic [3:0] part gets used for a, b, and c. Our actual parsing routines (see sv-ansi-portdecls) don't try to follow these rules. Instead, they give us a list of "raw" vl-parsed-ansi-port-p structures, which we then need to convert into actual ports, port declarations, and variable declarations.
