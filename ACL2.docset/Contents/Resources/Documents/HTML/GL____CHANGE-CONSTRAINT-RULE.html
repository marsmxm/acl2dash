A copying macro that lets you create new constraint-rule-p structures, based on existing structures.

Syntax:

(change-constraint-rule x 
                        [:thmname <thmname>] 
                        [:lit-alist <lit-alist>] 
                        [:syntaxp <syntaxp>])

This is a sometimes useful alternative to make-constraint-rule. It constructs a new constraint-rule-p structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by std::defaggregate.

Macro: change-constraint-rule

(defmacro
  change-constraint-rule (x &rest args)
  (std::change-aggregate 'constraint-rule
                         x args
                         '((:thmname . constraint-rule->thmname)
                           (:lit-alist . constraint-rule->lit-alist)
                           (:syntaxp . constraint-rule->syntaxp))
                         'change-constraint-rule
                         'nil))
