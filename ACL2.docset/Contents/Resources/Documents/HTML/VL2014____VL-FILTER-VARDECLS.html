Partition a list of vl-vardecls by name.

Signature
(vl-filter-vardecls names x) → (mv named unnamed)
Arguments
names — Names to filter with.
    Guard (string-listp names).
x — List to filter.
    Guard (vl-vardecllist-p x).

The only reason to use this function is efficiency. Logically, named is equal to vl-keep-vardecls and unnamed is equal to vl-delete-vardecls. We leave this function enabled and would think it odd to ever prove a theorem about it.

Definitions and Theorems

Function: vl-filter-vardecls

(defun
 vl-filter-vardecls (names x)
 (declare (xargs :guard (and (string-listp names)
                             (vl-vardecllist-p x))))
 (let
  ((__function__ 'vl-filter-vardecls))
  (declare (ignorable __function__))
  (mbe :logic (mv (vl-keep-vardecls names x)
                  (vl-delete-vardecls names x))
       :exec (b* (((when (atom names))
                   (mv nil (list-fix x)))
                  ((when (atom x)) (mv nil nil))
                  (fal (make-lookup-alist names))
                  ((local-stobjs nrev nrev2)
                   (mv yes no nrev nrev2))
                  ((mv nrev nrev2)
                   (vl-fast-filter-vardecls names fal x nrev nrev2))
                  (- (fast-alist-free fal))
                  ((mv yes nrev) (nrev-finish nrev))
                  ((mv no nrev2) (nrev-finish nrev2)))
                 (mv yes no nrev nrev2)))))
