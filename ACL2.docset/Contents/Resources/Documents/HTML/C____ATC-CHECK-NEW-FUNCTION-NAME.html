Check that a C function name is new.

Signature
(atc-check-new-function-name fn-name prec-fns) 
   
(mv okp conflicting-fn)
Arguments
fn-name Guard (stringp fn-name).
prec-fns Guard (atc-symbol-fninfo-alistp prec-fns).
Returns
okp Type (booleanp okp).
conflicting-fn Type (symbolp conflicting-fn).

That is, ensure that the symbol name of fn differs from the ones in prec-fns. It is not enough that the symbols are different: the symbol names must be different, because package names are ignored when translating to C. We return a boolean saying whether the check succeeds or not. If it does not, we return the function that causes the conflict, i.e. that has the same symbol name as fn.

Function: atc-check-new-function-name-aux

(defun
 atc-check-new-function-name-aux
 (fn-name fns)
 (declare (xargs :guard (and (stringp fn-name)
                             (symbol-listp fns))))
 (let
    ((__function__ 'atc-check-new-function-name-aux))
    (declare (ignorable __function__))
    (cond ((endp fns) (mv t nil))
          ((equal (symbol-fix fn-name)
                  (symbol-name (symbol-fix (car fns))))
           (mv nil (car fns)))
          (t (atc-check-new-function-name-aux fn-name (cdr fns))))))

Function: atc-check-new-function-name

(defun
  atc-check-new-function-name
  (fn-name prec-fns)
  (declare (xargs :guard (and (stringp fn-name)
                              (atc-symbol-fninfo-alistp prec-fns))))
  (let ((__function__ 'atc-check-new-function-name))
       (declare (ignorable __function__))
       (atc-check-new-function-name-aux
            fn-name
            (strip-cars (atc-symbol-fninfo-alist-fix prec-fns)))))
