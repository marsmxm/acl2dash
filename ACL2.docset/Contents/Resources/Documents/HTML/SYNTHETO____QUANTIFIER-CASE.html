Case macro for the different kinds of quantifier structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a quantifier structure, or to split into cases based on its type.

Short Form

In its short form, quantifier-case allows you to safely check the type of a quantifier structure. For example:

(quantifier-case x :forall)

is essentially just a safer alternative to writing:

(equal (quantifier-kind x) :forall)

Why is using quantifier-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :forall is a valid kind of quantifier structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :forall to something else. It also means you get no help if you just make a typo when writing the :forall symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, quantifier-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(quantifier-case x
  :forall ...
  :exists ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :forall case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a forall b* binder.
