Modifying constructor for expression-result-ok structures.

Syntax
(change-expression-result-ok x 
                             [:get <get>])

This is an often useful alternative to make-expression-result-ok.

We construct a new expression-result-ok structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-expression-result-ok

(defmacro
     change-expression-result-ok
     (x &rest args)
     (std::change-aggregate 'expression-result-ok
                            x args
                            '((:get . expression-result-ok->get))
                            'change-expression-result-ok
                            'nil))
