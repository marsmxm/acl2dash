Modifying constructor for vl-propclock structures.

Syntax
(change-vl-propclock x 
                     [:trigger <trigger>] 
                     [:then <then>])

This is an often useful alternative to make-vl-propclock.

We construct a new vl-propclock structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by defprod.

Macro: change-vl-propclock

(defmacro
     change-vl-propclock (x &rest args)
     (std::change-aggregate 'vl-propclock
                            x args
                            '((:trigger . vl-propclock->trigger)
                              (:then . vl-propclock->then))
                            'change-vl-propclock
                            'nil))
