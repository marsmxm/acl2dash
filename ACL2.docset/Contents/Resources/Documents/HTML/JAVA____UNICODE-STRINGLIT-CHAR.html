Fixtype of Unicode input characters usable in string literals.

This corresponds to the first alternative of the grammar rule for string-character [JLS:3.10.5], which is input-character except for double quote and backslash, where input-character is defined in [JLS:3.4] as a unicode-input-character except for CR and LF.

Since unicode-input-character is captured by unicode-input-char-p in our formalization, we define a Unicode character for a string literal (as conveyed by the stringlit part of the type and function names) as a Unicode input character that is not CR, LF, double quote, and backslash.

Function: unicode-stringlit-char-p

(defun unicode-stringlit-char-p (x)
       (declare (xargs :guard t))
       (let ((__function__ 'unicode-stringlit-char-p))
            (declare (ignorable __function__))
            (and (unicode-input-char-p x)
                 (not (member (unicode-input-char->unicode x)
                              (list (char-code #\Return)
                                    (char-code #\Newline)
                                    (char-code #\")
                                    (char-code #\\)))))))

Function: unicode-stringlit-char-fix

(defun unicode-stringlit-char-fix (x)
       (declare (xargs :guard (unicode-stringlit-char-p x)))
       (mbe :logic (if (unicode-stringlit-char-p x)
                       x
                       (make-unicode-input-char :unicode 0
                                                :umarker 0))
            :exec x))
