Add or delete an error output string from the inhibit-er-soft-table

See set-inhibit-er-soft for relevant background.

General Form:
(toggle-inhibit-er-soft string)

where string is the name of some error output like "Translate" or "Failure".

Note: This is an event! It does not print the usual event summary but nevertheless changes the ACL2 logical world and is so recorded. It is local to the book or encapsulate form in which it occurs; see toggle-inhibit-er-soft! for a corresponding non-local event. Indeed, (toggle-inhibit-er-soft str) is equivalent to (local (toggle-inhibit-er-soft! str)).

The given string is added to the list of strings used for inhibiting error output if it is not there already and is deleted from the list if it is there. Case is unimportant in string. See set-inhibit-er-soft.
