Modifying constructor for integer-result-ok structures.

Syntax
(change-integer-result-ok x 
                          [:get <get>])

This is an often useful alternative to make-integer-result-ok.

We construct a new integer-result-ok structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-integer-result-ok

(defmacro
     change-integer-result-ok (x &rest args)
     (std::change-aggregate 'integer-result-ok
                            x
                            args '((:get . integer-result-ok->get))
                            'change-integer-result-ok
                            'nil))
