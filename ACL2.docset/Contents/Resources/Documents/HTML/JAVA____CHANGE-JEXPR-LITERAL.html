Modifying constructor for jexpr-literal structures.

Syntax
(change-jexpr-literal x 
                      [:get <get>])

This is an often useful alternative to make-jexpr-literal.

We construct a new jexpr-literal structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-jexpr-literal

(defmacro
     change-jexpr-literal (x &rest args)
     (std::change-aggregate 'jexpr-literal
                            x args '((:get . jexpr-literal->get))
                            'change-jexpr-literal
                            'nil))
