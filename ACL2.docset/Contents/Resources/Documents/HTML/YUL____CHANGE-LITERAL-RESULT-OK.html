Modifying constructor for literal-result-ok structures.

Syntax
(change-literal-result-ok x 
                          [:get <get>])

This is an often useful alternative to make-literal-result-ok.

We construct a new literal-result-ok structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by fty::defprod.

Macro: change-literal-result-ok

(defmacro
     change-literal-result-ok (x &rest args)
     (std::change-aggregate 'literal-result-ok
                            x
                            args '((:get . literal-result-ok->get))
                            'change-literal-result-ok
                            'nil))
