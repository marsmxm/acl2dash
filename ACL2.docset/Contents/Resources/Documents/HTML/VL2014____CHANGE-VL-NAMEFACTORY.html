Modifying constructor for vl-namefactory structures.

Syntax
(change-vl-namefactory x 
                       [:mod <mod>] 
                       [:namedb <namedb>])

This is an often useful alternative to make-vl-namefactory.

We construct a new vl-namefactory structure that is a copy of x, except that you can explicitly change some particular fields. Any fields you don't mention just keep their values from x.

Definition

This is an ordinary change- macro introduced by defprod.

Macro: change-vl-namefactory

(defmacro
     change-vl-namefactory (x &rest args)
     (std::change-aggregate 'vl-namefactory
                            x args
                            '((:mod . vl-namefactory->mod)
                              (:namedb . vl-namefactory->namedb))
                            'change-vl-namefactory
                            'remake-vl-namefactory))
