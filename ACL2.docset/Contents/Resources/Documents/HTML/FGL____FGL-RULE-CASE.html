Case macro for the different kinds of fgl-rule structures.

This is an ACL2::fty sum-type case macro, typically introduced by defflexsum or fty::deftagsum. It allows you to safely check the type of a fgl-rule structure, or to split into cases based on its type.

Short Form

In its short form, fgl-rule-case allows you to safely check the type of a fgl-rule structure. For example:

(fgl-rule-case x :rewrite)

is essentially just a safer alternative to writing:

(equal (fgl-rule-kind x) :rewrite)

Why is using fgl-rule-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :rewrite is a valid kind of fgl-rule structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :rewrite to something else. It also means you get no help if you just make a typo when writing the :rewrite symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, fgl-rule-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(fgl-rule-case x
  :rewrite ...
  :primitive ...
  :meta ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :rewrite case, you can use defprod-style foo.bar style accessors for x without having to explicitly add a rewrite b* binder.
