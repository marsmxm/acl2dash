Case macro for the different kinds of string-literal-char structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a string-literal-char structure, or to split into cases based on its type.

Short Form

In its short form, string-literal-char-case allows you to safely check the type of a string-literal-char structure. For example:

(string-literal-char-case x :char)

is essentially just a safer alternative to writing:

(equal (string-literal-char-kind x) :char)

Why is using string-literal-char-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :char is a valid kind of string-literal-char structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :char to something else. It also means you get no help if you just make a typo when writing the :char symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, string-literal-char-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(string-literal-char-case x
  :char ...
  :escape ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :char case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a char b* binder.
