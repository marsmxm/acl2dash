Case macro for the different kinds of vl-genelement structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or deftagsum. It allows you to safely check the type of a vl-genelement structure, or to split into cases based on its type.

Short Form

In its short form, vl-genelement-case allows you to safely check the type of a vl-genelement structure. For example:

(vl-genelement-case x :vl-genloop)

is essentially just a safer alternative to writing:

(equal (vl-genelement-kind x) :vl-genloop)

Why is using vl-genelement-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :vl-genloop is a valid kind of vl-genelement structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :vl-genloop to something else. It also means you get no help if you just make a typo when writing the :vl-genloop symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, vl-genelement-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(vl-genelement-case x
  :vl-genloop ...
  :vl-genif ...
  :vl-gencase ...
  :vl-genblock ...
  :vl-genarray ...
  :vl-genbase ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :vl-genloop case, you can use defprod-style foo.bar style accessors for x without having to explicitly add a vl-genloop b* binder.
