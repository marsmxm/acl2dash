Recognizer for an vl-exprtype-p or nil.

This is an option type introduced by fty::defoption. Note that defoption is just a wrapper for fty::defflexsum, so there are :none and :some member types, a case macro, and so forth.

Member Types
:none → vl-maybe-exprtype-none
Represents that no vl-maybe-exprtype is available, i.e., Nothing or None.
:some → vl-maybe-exprtype-some
An available vl-maybe-exprtype, i.e., Just val or Some val.

We use this for the finaltype fields in our expressions. It allows us to represent expressions whose types have not yet been computed.
