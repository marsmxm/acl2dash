Generate the operations for an array member of the structures defined by defstruct.

Signature

                                struct-tag-fix name type size)

(mv event checkers readers writers 
    reader-return-thms writer-return-thms)
Arguments

struct-tag-p — Guard (symbolp struct-tag-p).

Returns

reader-return-thms — Type (symbol-listp reader-return-thms).
writer-return-thms — Type (symbol-listp writer-return-thms).

This is only for integer arrays. The type parameter of this ACL2 function is the type of the elements of the array.

The readers and writers read and write elements of the array, not the whole array, unlike for integer members. However, these operations are internally defined as a combination of reading/writing the whole arrays and reading/writing their elements.

We generate local lemmas about the writing of the whole arrays yielding results of the expected types that in particular are not errors. In fact, the generated (readers and) writers never return errors.

Similarly to the operations on integer arrays in atc-arrays, we generate 11 readers and 11 writers, which only differ in the type of the index, namely ACL2 integers or (10 types of) C integers; the latter are defined in terms of the former.

Besides readers and writers, we also generate 11 checkers, which check that the index is within the array bounds. These are used in the guards of the readers and writers, again analogously to the operations on integer arrays in atc-arrays. One is for ACL2 integers, and the other 10 are for C integers, with again the latter defined in terms of the form.

The reason for generating the index checkers, as opposed to using the existing ones on integer arrays, is that, like for readers as explained above, we do not provide operations to read the whole array, but rather its elements. Thus, it makes sense that the checking functions are also provided here. Note that the definition of the checkers is in terms of the known length of the array member, and that guard verification for functions involving these structures is performed by opening the checkers.
