Case macro for the different kinds of type-argument structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a type-argument structure, or to split into cases based on its type.

Short Form

In its short form, type-argument-case allows you to safely check the type of a type-argument structure. For example:

(type-argument-case x :reftype)

is essentially just a safer alternative to writing:

(equal (type-argument-kind x) :reftype)

Why is using type-argument-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :reftype is a valid kind of type-argument structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :reftype to something else. It also means you get no help if you just make a typo when writing the :reftype symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, type-argument-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(type-argument-case x
  :reftype ...
  :wildcard ...
  :wildcard-extends ...
  :wildcard-super ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :reftype case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a reftype b* binder.
