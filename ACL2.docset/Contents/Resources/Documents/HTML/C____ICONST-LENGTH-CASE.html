Case macro for the different kinds of iconst-length structures.

This is an ACL2::fty sum-type case macro, typically introduced by fty::defflexsum or fty::deftagsum. It allows you to safely check the type of a iconst-length structure, or to split into cases based on its type.

Short Form

In its short form, iconst-length-case allows you to safely check the type of a iconst-length structure. For example:

(iconst-length-case x :none)

is essentially just a safer alternative to writing:

(equal (iconst-length-kind x) :none)

Why is using iconst-length-case safer? When we directly inspect the kind with equal, there is no static checking being done to ensure that, e.g., :none is a valid kind of iconst-length structure. That means there is nothing to save you if, later, you change the kind keyword for this type from :none to something else. It also means you get no help if you just make a typo when writing the :none symbol. Over the course of developing VL, we found that such issues were very frequent sources of errors!

Long Form

In its longer form, iconst-length-case allows you to split into cases based on the kind of structure you are looking at. A typical example would be:

(iconst-length-case x
  :none ...
  :long ...
  :llong ...)

It is also possible to consolidate ``uninteresting'' cases using :otherwise.

For convenience, the case macro automatically binds the fields of x for you, as appropriate for each case. That is, in the :none case, you can use fty::defprod-style foo.bar style accessors for x without having to explicitly add a none b* binder.
